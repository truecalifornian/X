Search Parameters Reference

Master all available X.COM search parameters to create powerful and precise searches.

Engagement Filters

Parameter Description Example
min_faves: Minimum number of likes min_faves:10
min_retweets: Minimum number of retweets min_retweets:5
min_replies: Minimum number of replies min_replies:3

Content Filters

Parameter Description Example
-filter:links Exclude posts with links -filter:links
-filter:replies Exclude reply posts -filter:replies
-filter:images Exclude posts with images -filter:images
filter:verified Only from verified accounts filter:verified

User Filters

from:username - Posts from specific user
to:username - Posts mentioning specific user
@username - Posts mentioning specific user (alternative)

Time-based Filters

since:2024-01-01 - Posts after specific date
until:2024-12-31 - Posts before specific date

Troubleshooting

Common issues and solutions when working with XSearch Pro.

Common Issues

üîç No Results Found

Problem: Your search returns no results or very few results.

Solution: Try these steps:

  • Lower your engagement thresholds (min_faves, min_retweets)
  • Remove some filters to broaden your search
  • Check if your search term is too specific
  • Verify that time-based filters aren't too restrictive

‚ö†Ô∏è Too Many Results

Problem: Your search returns an overwhelming number of results.

Solution: Try these approaches:

  • Increase engagement thresholds
  • Add more specific filters
  • Use time-based filters to focus on recent content
  • Include user-specific filters to target key accounts

üîó URL Encoding Issues

Problem: Search URLs don't work correctly in X.COM.

Solution: XSearch Pro automatically handles URL encoding, but if you encounter issues:

  • Use the copy URL button for proper encoding
  • Avoid manually editing generated URLs
  • Check for special characters in search terms
